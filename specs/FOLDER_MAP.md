# Folder: Specifications (specs)

## Overview
This directory contains the historical and active feature specifications, technical plans, and task breakdowns generated by the **Speckit** workflow. It provides a trace of "What" was built and "How" it was implemented.

## Subdirectories

### [003-codebase-bug-fixes/](./003-codebase-bug-fixes/FOLDER_MAP.md)
- **Status**: Completed / Reference.
- **Purpose**: A comprehensive audit and bug-fix sprint addressing core stability and dependency issues in April 2026.

### [004-ui-enhancement-bug-fixes/](./004-ui-enhancement-bug-fixes/FOLDER_MAP.md)
- **Status**: Active / Recent.
- **Purpose**: Improvements to the "Cyber-Luxury" UI, resolving visual regressions in the Lobby and Profile components.

## Standard Spec Artifacts
Within each sub-directory, you will typically find:
- **spec.md**: Functional requirements and user stories.
- **plan.md**: Technical design and architectural decisions.
- **tasks.md**: The granular TODO list used during implementation.
- **research.md**: Exploration and analysis notes.
- **data-model.md**: Updates or descriptions of Firestore schema changes.
